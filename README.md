## 📚 What's This?

An organized, reusable toolkit of AI primitives to supercharge your AI coding harness. Each primitive is crafted for a specific task with clear instructions and use cases, authored once under [`.apm/`](.apm/) and installable across GitHub Copilot, Claude Code, Cursor, OpenCode, Gemini and Windsurf via [APM](https://microsoft.github.io/apm/).

## 🤝 Bundled agents & self-consumption

Curated agents from [`github/awesome-copilot`](https://github.com/github/awesome-copilot) are declared as SHA-pinned APM dependencies in [`apm.yml`](apm.yml). This repo also **self-consumes** them: `apm install` (pinned to the `copilot` target) materializes them into [`.github/agents/`](.github/agents/), which is committed **on purpose**.

That committed snapshot is a **review gate**: bumping a dependency (`apm update`) and re-running `apm install` surfaces the real agent-text change as a diff in the pull request — so a malicious or prompt-hijacking edit can't slip in behind a bare SHA bump. The rest of the `apm install` output (skills, instructions, hooks, deployed prompt copies) is trivially regenerable from `.apm/` and is git-ignored, so the only intentional duplication is `.github/agents/`. `.apm/` remains the source of truth.

## 📦 Install via APM

This toolkit is an [APM (Agent Package Manager)](https://microsoft.github.io/apm/)
**producer**. Install its primitives as native commands in your AI harness
(GitHub Copilot, Claude Code, Cursor, OpenCode, Gemini, Windsurf).

Pin a released version and pick your harness with `--target`:

<!-- x-release-please-start-version -->
```bash
apm install DevSecNinja/ai-toolkit#v0.6.2 --target copilot
```
<!-- x-release-please-end -->

> `apm install` resolves by GitHub repo name (`ai-toolkit`). Omit `#v0.1.0` to
> install from the latest default branch, and omit `--target` to install into
> every detected harness.

Run `apm targets` to list the supported harnesses you can pass to `--target`
(e.g. `copilot`, `claude`, `cursor`, `opencode`, `gemini`, `windsurf`).

See the [APM Producer Guide](/docs/apm.md) for how these primitives are authored,
produced and consumed.

## 💡 Tips for Writing Great Prompts

- **Be Specific** - Clear instructions yield better results
- **Set Context** - Define the role and expertise level
- **Structure Output** - Request specific formats or sections
- **Iterate** - Refine prompts based on results
